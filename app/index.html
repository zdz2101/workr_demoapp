workr · live demo a very simple R data pipeline — 1.1.0 features, running in your browser workr repo ↗

A workflow is an ordered list of function calls in YAML. RunProject() runs a folder of them in phases, carrying each phase's output to the next. Press Run all to watch it execute — the tables on the right are real output computed with workr, not mock-ups. Click any card for its YAML and steps.

Same pipeline also runs headless in CI: a two-job GitHub Actions workflow saves the outputs as an artifact on one run and restores them on the next via workr's built-in github_artifact hook (the "Storage hook" selector above).

MakeWorkflowList() discovers workflows in a folder, sorts them by meta$Priority, and skips any marked Active: false. Flip the switch to include the work-in-progress ZScore workflow.
